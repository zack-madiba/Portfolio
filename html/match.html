1. Collecte des Données

Extraction sécurisée depuis Google Sheets grace aux bibliothèques Python requests, BytesIO, Pandas, sans stockage intermédiaire sur disque.

python 
import pandas as pd
import requests
from io import BytesIO

url = "https://docs.google.com/spreadsheets/d/1-ozQDxIBwo-XhgNrfqDZaPwlN9j88c8E/export?format=xlsx"
save_path = "data.csv"

def download_file(url, save_path):
    try:
        response = requests.get(url)
        if response.status_code == 200:
            df = pd.read_excel(BytesIO(response.content))
            df.to_csv(save_path, index=False)
            print(f"Fichier téléchargé avec succès dans {save_path}")
            return df
    except requests.exceptions.RequestException as e:
        print(f"Erreur: {e}")
Fonction de téléchargement sécurisé depuis Google Sheets
Sécurité: L'utilisation de BytesIO permet de manipuler les données en mémoire sans risque d'exposition sur le disque.

2. Nettoyage & Transformation

Standardisation des formats, gestion des valeurs manquantes et création de champs dérivés pour l'analyse.

python 
def nettoyer_donnees_investisseurs(save_path):
    df_raw = pd.read_csv(save_path)
    df_cleaned = df_raw.copy()
    
    # Nettoyage des colonnes combinées
    columns_raw = ["Investisseur.statut", "Statut match", "Typologie"]
    for col in columns_raw:
        df_cleaned[col] = df_cleaned[col].astype(str).str.split(' ', n=1).str[1]
    
    # Conversion des montants (K€ → €)
    df_cleaned["Montant estimé"] = (
        df_cleaned["Montant estimé"]
        .replace({r"[Kk]": ""}, regex=True)
        .astype(float) * 1000
    )
    
    # Création d'un contact unique
    df_cleaned['contact investisseur'] = df_cleaned.apply(
        lambda row: f"{row['Nom CP']} {row['Prénom']}" 
        if pd.notna(row['Nom CP']) && pd.notna(row['Prénom']) 
        else row['Client Principal'] if pd.notna(row['Client Principal']) 
        else row['Investisseur'],
        axis=1
    )
    
    return df_cleaned
Fonction de nettoyage et transformation des données
Optimisation: Utilisation de vectorisation avec Pandas pour des opérations plus efficaces que les boucles.

4. Analyse des Données

Requêtes SQL ciblées pour répondre aux questions métier et identifier des opportunités.

Investisseurs les Plus Actifs

sql 

Select i.investisseur as "Nom d'investisseur",
t.typologie as "Type d'investisseur",
count(*) as "Occurences"
from investisseur i
join match m on m.investisseur_ID = i.ID_investisseur
join typologie t on t.id_typologie = m.typologie_id
join statut_match s on s.ID_statut_match = m.statut_match_id
WHERE s.statut_match IN ('En RDV', 'Deal', 'Potentiel', 'Dossier envoyé')
group by i.investisseur, t.typologie
order by "occurences" DESC
limit 5;
Requête d'analyse des investisseurs les plus actifs
le top 5 des investisseurs actifs montre que les investisseurs sont à égalité avec seulement 2 rencontres maximum, ce qui montre un niveau d'activité très faible sur la plateforme. Les Business Angels (4/5) dominent les rencontres effectives (8/10), confirmant qu'ils sont plus accessibles que les gros fonds

Identification des investisseurs clés à cibler en priorité pour de nouvelles startups, avec un taux de réponse élevé.

Matchs à Relancer par Typologie

sql 

SELECT
t.typologie,
COUNT(m.id_match) AS matchs_a_relancer
FROM match m
JOIN typologie t ON m.typologie_id = t.id_typologie
JOIN statut_match s ON m.statut_match_id = s.ID_statut_match
WHERE s.statut_match IN ('relancer', 'Intérêt', 'Intention', 'A présenter', 'Stand by','Potentiel')
GROUP BY t.typologie
ORDER BY matchs_a_relancer DESC;
Requête d'analyse des opportunités à relancer
Les investisseurs les plus actifs en nombre de matchs à relancer sont majoritairement issus des typologies BA et VC, avec une légère prédominance des Business Angels. Les investisseurs financent majotitairement des projets jeunes, à fort potentiel mais encore risqués, ce qui en fait des cibles prioritaires pour une relance stratégique et structurée
Optimisation: Exclusion des statuts non définis, car non pertinents pour concentrer les efforts sur les opportunités actionnables.
sql 

SELECT t.typologie,
COUNT(m.id_match) AS matchs_a_relancer
